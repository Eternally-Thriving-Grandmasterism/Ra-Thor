# self-improvement-extensions

TOLC + 7 Living Mercy Gates evaluation module for Rathor.ai self-evolution.

This crate provides robust evaluation of proposals generated by the self-improvement loop, using both TOLC principles and the 7 Living Mercy Gates.

## Features

- Strong JSON extraction from LLM responses (4-strategy fallback)
- Comprehensive TOLC scoring (Truth, Order, Logic, Compassion)
- Full 7 Living Mercy Gates scoring
- Derived averages and acceptance logic
- Output validation
- Structured logging (info + trace)
- Automated integration tests

## TOLC Scoring

TOLC (True Original Lord Creator) provides four foundational scoring dimensions:

- **Truth**: Is the proposal based on verifiable facts and honest assessment?
- **Order**: Does it increase healthy structure and reduce unnecessary complexity?
- **Logic**: Is the reasoning internally consistent with no contradictions?
- **Compassion**: Does it consider the well-being and impact on affected beings?

### Code Example: Accessing TOLC Scores

```rust
use self_improvement_extensions::evaluate_proposal_with_tolc_and_mercy;

let result = evaluate_proposal_with_tolc_and_mercy(&model, proposal_text);

println!("TOLC Scores:");
println!("  Truth:      {:.1}", result.truth_score);
println!("  Order:      {:.1}", result.order_score);
println!("  Logic:      {:.1}", result.logic_score);
println!("  Compassion: {:.1}", result.compassion_score);

println!("Average TOLC Score: {:.2}", result.average_tolc_score);
```

### Code Example: Using TOLC in Decision Logic

```rust
if result.average_tolc_score >= 7.0 && result.is_acceptable() {
    println!("Proposal has strong TOLC alignment and passes Mercy gates.");
} else {
    println!("Proposal needs improvement in TOLC or Mercy dimensions.");
}
```

## Key Types

### `EvaluationResult`

Contains individual TOLC and Mercy Gate scores, averages, and a final `passes_threshold` from the LLM.

### `is_acceptable()`

```rust
pub fn is_acceptable(&self) -> bool
```

Returns `true` only if:
- The LLM reports `passes_threshold = true`, **and**
- `sovereignty_gate_score >= 7.0`, **and**
- `non_harm_gate_score >= 6.0`, **and**
- `harmony_gate_score >= 6.0`

## Usage

```rust
use self_improvement_extensions::evaluate_proposal_with_tolc_and_mercy;

let result = evaluate_proposal_with_tolc_and_mercy(&model, proposal_text);

if result.is_acceptable() {
    // Proposal passes mercy and TOLC gates
}
```

## Building & Testing

From the repository root:

```bash
# Basic check
cargo check -p self-improvement-extensions

# With optional LLM backend
cargo check -p self-improvement-extensions --features llama-cpp

# Run unit + integration tests
cargo test -p self-improvement-extensions
```

**Note:** The `llama-cpp` feature pulls in an experimental local dependency. The core evaluation and JSON extraction logic works without it.
