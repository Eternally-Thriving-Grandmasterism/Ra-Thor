# self-improvement-extensions

TOLC + 7 Living Mercy Gates evaluation module for Rathor.ai self-evolution.

This crate provides robust evaluation of proposals generated by the self-improvement loop, using both TOLC principles and the 7 Living Mercy Gates.

## Features

- Strong JSON extraction from LLM responses (4-strategy fallback)
- Comprehensive TOLC scoring (Truth, Order, Logic, Compassion)
- Full 7 Living Mercy Gates scoring
- Derived averages and acceptance logic
- Unit tests for extraction robustness

## Key Types

### `EvaluationResult`

Contains individual scores, averages, and a final `passes_threshold` from the LLM.

### `is_acceptable()`

```rust
pub fn is_acceptable(&self) -> bool
```

Returns `true` only if:
- The LLM reports `passes_threshold = true`, **and**
- `sovereignty_gate_score >= 7.0`, **and**
- `non_harm_gate_score >= 6.0`, **and**
- `harmony_gate_score >= 6.0`

This ensures proposals meet meaningful mercy standards, with Sovereignty held to the highest bar.

## Usage

```rust
use self_improvement_extensions::evaluate_proposal_with_tolc_and_mercy;

let result = evaluate_proposal_with_tolc_and_mercy(&model, proposal_text);

if result.is_acceptable() {
    // Proposal passes mercy and TOLC gates
}
```

## Building & Testing

From the repository root:

```bash
# Basic check
cargo check -p self-improvement-extensions

# With optional LLM backend
cargo check -p self-improvement-extensions --features llama-cpp

# Run unit tests
cargo test -p self-improvement-extensions
```

**Note:** The `llama-cpp` feature pulls in an experimental local dependency. The core evaluation and JSON extraction logic works without it.

## Dependencies

- `serde_json` (via workspace)
- `regex`
- Optional: `llama-cpp-gguf` (behind `llama-cpp` feature)
