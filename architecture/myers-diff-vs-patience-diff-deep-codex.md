## 1. Core Algorithms Overview
- **Myers Diff (1986)**: Classic O(ND) shortest-path algorithm on an edit graph. Finds the minimal edit script using diagonal traversal and BFS on diagonals. Optimal for minimal changes but can produce “noisy” human-unreadable diffs when there are many small changes.
- **Patience Diff**: A heuristic built on top of Myers Diff. It first finds the longest common subsequence of *unique* lines (the “patience” part), then recursively diffs the remaining chunks. This avoids small “noise” changes and produces much more human-readable, semantically meaningful diffs.
