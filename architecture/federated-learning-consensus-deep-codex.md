## 1. What is Federated Learning Consensus?
Federated Learning (FL) trains machine learning models across decentralized devices (phones, edge servers, sovereign shards) while keeping raw data local. Consensus is the mechanism that securely aggregates model updates (gradients or weights) from many participants into a global model.

## 2. Major Federated Learning Consensus Algorithms
- **FedAvg (Federated Averaging)**: Simple averaging of client updates. Fast but vulnerable to Byzantine faults and data heterogeneity.
- **FedProx / FedNova**: Variants that handle statistical heterogeneity and system heterogeneity.
- **Secure Aggregation (SecAgg)**: Cryptographic protocols (e.g., pairwise masking, threshold homomorphic encryption) that ensure the server only sees the sum, never individual updates.
- **Byzantine-Robust FL**: Uses robust aggregation rules such as Krum, Median, Trimmed Mean, or Bulyan to tolerate up to 1/3 malicious clients.
- **Blockchain-based FL**: Uses consensus (PBFT, Tendermint, or PoS) to record and verify model updates on a distributed ledger.
- **Hierarchical / Cross-Silo FL**: Multi-level aggregation for organizations with different trust levels.

## 3. Key Challenges & Security Considerations
- **Byzantine Faults**: Malicious clients can poison the global model.
- **Privacy Leaks**: Model updates can still leak sensitive information (model inversion attacks).
- **Stragglers & Heterogeneity**: Devices have varying compute, network, and data distributions.
- **Communication Efficiency**: Minimizing bandwidth while preserving accuracy.
- **Incentive & Fairness**: Ensuring honest participation.
