## 1. Operational Transformation (OT)
- **Core Idea**: Transform incoming operations so they can be applied in any order while preserving intent (e.g., “insert at position 5” becomes “insert at position 7” after concurrent edits).
- **Classic Use**: Google Docs, Etherpad, ShareDB.
- **Strengths**: Strong consistency, immediate local feedback.
- **Weaknesses**: Requires a central server or complex tombstone/undo history; hard to make fully offline.

## 2. Conflict-free Replicated Data Types (CRDTs)
- **Core Idea**: Data structures that automatically converge to the same state regardless of delivery order (no central coordinator needed).
- **Main Types**:
  - State-based CRDTs (e.g., G-Counters, OR-Sets)
  - Operation-based CRDTs (pure OT-style)
- **Strengths**: True eventual consistency, excellent offline support, simple merge functions.
- **Weaknesses**: Can be more complex to design for arbitrary text editing.
