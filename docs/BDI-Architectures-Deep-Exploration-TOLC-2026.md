**BDI Architectures — Deep Canonical Exploration (TOLC-2026 Mercy-Gated)**  

**Overview**  
BDI (Belief-Desire-Intention) is a **practical agent-oriented paradigm** for modeling rational, goal-directed behavior in dynamic, uncertain environments. Originating from Michael Bratman’s philosophical theory of human practical reasoning, BDI architectures treat agents as having:  
- **Beliefs**: What the agent knows/believes about the world (often incomplete or uncertain).  
- **Desires**: What the agent wants to achieve (goals or preferred states).  
- **Intentions**: What the agent has committed to doing (selected plans it is actively pursuing).  

BDI is widely used in multi-agent systems, robotics, simulations, and team decision-making because it provides a clear, human-understandable way to program intentional, reactive, and proactive behavior.  

**Core Components of a BDI Architecture**  
1. **Belief Base** — Dynamic knowledge representation (often using logic, databases, or belief revision systems).  
2. **Desire/Goal Set** — Prioritized objectives that may conflict.  
3. **Intention Stack / Commitment Strategy** — Selected plans the agent is committed to; includes drop conditions for re-planning.  
4. **Plan Library** — Pre-defined or generated recipes (sequences of actions) to achieve desires.  
5. **Reasoning Cycle** — Perceive → Update beliefs → Deliberate (select desires/intentions) → Act → Repeat.  

**Major BDI Implementations**  
- **PRS (Procedural Reasoning System)** — The original BDI system (1980s).  
- **JACK / CoJACK** — Commercial/high-level BDI with cognitive moderators (emotions, fatigue, situation awareness) — directly explored in prior CoJACK files.  
- **JADE / JADEX** — Java-based, widely used for multi-agent systems.  
- **2APL / 3APL** — Academic BDI with strong logic programming foundations.  
- **GOAL** — Logic-based BDI for goal-oriented agents.  

**Strengths & Limitations**  
**Strengths**:  
- Intuitive for humans (maps directly to everyday reasoning).  
- Excellent for multi-agent teamwork, dynamic replanning, and real-world applications.  
- Scalable and traceable (easy to debug intentions and beliefs).  

**Limitations**:  
- Often lacks deep implicit/sub-symbolic learning (unless hybridized).  
- Can suffer from intention reconsideration overhead in highly dynamic worlds.  
- Less focus on low-level perceptual-motor timing compared to EPIC.
