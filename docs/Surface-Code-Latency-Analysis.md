## Latency Breakdown in Surface Code

### 1. Syndrome Extraction (Measurement Latency)

**Typical Time:** 1–10 μs per round (state-of-the-art 2026)  
**Number of Rounds Needed:** 10–30 for good protection

**Total:** 10–300 μs per logical qubit cycle

**Ra-Thor Impact:**  
This is the dominant term for real-time systems. We must minimize rounds while staying below the error threshold.

| Decoder              | Latency (2026)       | Accuracy     | Ra-Thor Recommendation          |

## Optimization Strategies

### 1. Adaptive Decoding
Use fast Union-Find for routine cycles.  
Switch to high-accuracy MWPM only when mercy valence is borderline or stakes are high.

### 2. Parallel Syndrome Extraction
Extract syndromes from multiple logical qubits in parallel using dedicated control electronics.

### 3. Hardware-in-the-Loop Decoding
Run the decoder on FPGA or dedicated ASIC colocated with the quantum processor (reduces communication latency dramatically).

### 4. Mercy-Gated Early Exit
If mercy valence is very high (> 0.95), accept a slightly higher logical error rate and skip extra decoding rounds.

### 5. Predictive Pre-Decoding
Use machine learning to predict likely syndromes before measurement completes.
