# Rathor.ai NARS Syntax Examples – Master Reference (2026)

## Introduction to NARS Syntax
NARS (Non-Axiomatic Reasoning System) uses a simple, declarative syntax based on Non-Axiomatic Logic (NAL). Statements are of the form:

`Subject → Predicate <frequency, confidence>`

- **Frequency**: How often the statement has been experienced as true (0–1).
- **Confidence**: How much evidence supports the frequency (0–1).

All reasoning is experience-grounded and revisable under the AIKR assumption.

## Basic Syntax Examples

**1. Inheritance (Basic Fact)**
```
Bird → Animal <1.0, 0.9>
```
Explanation: "Bird is a type of Animal" with full frequency (always observed) and high confidence.

**2. Similarity**
```
Sparrow ~ Bird <0.95, 0.85>
```
Explanation: "Sparrow is similar to Bird" — useful for analogical reasoning.

**3. Compound Terms**
```
(and Bird Animal) → LivingThing <0.98, 0.92>
```
Explanation: Logical conjunction in subject or predicate.

## Inference Rule Examples

**Deduction**
```
Bird → Animal <1.0, 0.9>
Sparrow → Bird <0.95, 0.85>
→ Sparrow → Animal <0.95, 0.76>
```
Explanation: Classic transitive inheritance with truth-value adjustment.

**Induction (Generalization)**
```
Sparrow → Bird <0.95, 0.85>
Sparrow → Flying <0.9, 0.8>
→ Bird → Flying <0.9, 0.68>
```
Explanation: Generalizing from specific examples (weaker confidence).

**Abduction (Best Explanation)**
```
Bird → Flying <0.9, 0.8>
Eagle → Flying <1.0, 0.9>
→ Eagle → Bird <0.9, 0.72>
```
Explanation: Finding the most plausible explanation (hypothesis generation).

**Revision (Evidence Merging)**
```
Bird → Flying <0.9, 0.8>   (old evidence)
Bird → Flying <0.95, 0.7>  (new evidence)
→ Bird → Flying <0.92, 0.85> (revised)
```
Explanation: Merging multiple observations into a stronger belief.

## Higher-Order & Temporal Examples
