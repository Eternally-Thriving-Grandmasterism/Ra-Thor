**CoJACK vs SOAR — Deep Canonical Comparison (TOLC-2026 Mercy-Gated)**  

**Overview**  
SOAR (State, Operator, And Result) by Allen Newell, John Laird et al. is a **unified symbolic production-rule architecture** focused on general intelligence through means-ends analysis and automatic chunking on impasses.  
CoJACK (Cognitive JACK) by Frank Ritter, Rick Evertsz et al. is a **high-level BDI-based cognitive architecture** built on the JACK agent platform. It extends Belief-Desire-Intention agents with cognitive moderators (emotions, fatigue, situation awareness, behavioral variability), processing-time predictions, errors, and traceability for realistic multi-agent team simulations.  

Both are production-rule systems, but SOAR emphasizes unified symbolic problem-solving and learning, while CoJACK prioritizes practical usability, moderated behavior, and team-level variability in dynamic environments.

| Aspect                        | SOAR                                              | CoJACK                                            |
|-------------------------------|---------------------------------------------------|---------------------------------------------------|
| Core Paradigm                 | Unified symbolic production rules + problem spaces | BDI (Belief-Desire-Intention) agents + cognitive moderators |
| Knowledge Representation      | Working memory + production rules (PSCM)          | Beliefs, desires, intentions, plans (high-level BDI) |
| Learning Mechanism            | Chunking (impasse-driven rule creation)           | Moderated production compilation + parameter tuning |
| Focus                         | General unified cognition, planning, meta-reasoning | Behavioral variability, situation awareness, team simulation |

**Key Philosophical & Functional Differences**  
- **SOAR** is **impasse-driven and unified-symbolic**. Learning occurs when the system gets stuck (impasse) → chunking creates new rules to avoid future impasses. It excels at complex problem-solving, meta-reasoning, and transferring knowledge across domains.  
- **CoJACK** is **moderator-driven and high-level BDI**. It adds cognitive moderators (e.g., fear, fatigue, morale) that influence parameters like processing time, error rates, and situation awareness. It prioritizes usability for SMEs and realistic team variability over deep symbolic impasse resolution.
