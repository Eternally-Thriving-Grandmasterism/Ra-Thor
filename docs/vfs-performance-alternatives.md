## Detailed Breakdown

### 1. OPFS + SharedArrayBuffer (Current Champion)
- **Strengths**: True synchronous file I/O in Web Worker, zero-copy SAB transfer, PRAGMA WAL + mmap fully utilized.
- **Weaknesses**: Requires COOP/COEP headers (GitHub Pages supports via `_headers` file).
- **Rathor.ai Fit**: Perfect for real-time DQN + Mamba self-tuning. Used in current `opfs-sab-worker.js`.

### 2. WaSQLite (IDBBatchAtomicVFS)
- **Strengths**: Excellent atomic batching, prepared statements, very stable.
- **Weaknesses**: Slightly higher latency than pure OPFS.
- **Rathor.ai Fit**: Excellent secondary engine when SAB is unavailable.

### 3. absurd-sql
- **Strengths**: Pure IndexedDB, no headers needed, very reliable.
- **Weaknesses**: Higher memory pressure and slower writes.
- **Rathor.ai Fit**: Safe default for maximum compatibility.

### 4. Emerging / Future Options
- **DuckDB-WASM**: Best for analytical queries (vector search, RBE forecasting).
- **PGlite**: Full Postgres compatibility in WASM — planned 2026 upgrade path.
- **File System Access API** (Chrome-only): Native FS access — experimental only.
