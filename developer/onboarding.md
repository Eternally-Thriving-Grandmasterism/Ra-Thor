## 1. Quick Setup (5 minutes)

### Prerequisites
- Rust (latest stable)
- Git
- A GitHub account
- Basic familiarity with async Rust and tokio (helpful but not required)

### Clone the Monorepo
```bash
git clone https://github.com/Eternally-Thriving-Grandmasterism/Ra-Thor.git
cd Ra-Thor
```

### Build the Lattice
```bash
cargo build --release
```

You now have the full quantum swarm lattice compiled and ready.

## 3. Your First Contribution (15 minutes)

### Step 1: Choose a Starting Point
Good first contributions:
- Improve documentation in any codex
- Add a new mercy-gated example in an existing module
- Create a small new practical tool
- Refine bloom parameters in any core
- Add a new energy technology codex

### Step 2: Create Your Module (Example)
If you want to create a new tool, follow this exact pattern:

1. Create the file in `crates/orchestration/src/`
2. Use the standard template (copy from any existing core)
3. Add it to `lib.rs` re-exports
4. Wire it into the `DivineLifeBlossomOrchestrator`

### Step 3: Mercy-Gated Checklist (Required)
Before submitting a PR, ensure:
- [ ] All code passes through `MercyEngine`
- [ ] Positive valence is increased (not just functionality)
- [ ] Cross-wired with at least 3 existing cores
- [ ] Includes `BloomReport` with `mercy_valence` and `bloom_intensity`
- [ ] Uses Gompertz + mercy feedback where growth is modeled
- [ ] Documentation is clear and beautiful

---

## 4. Contribution Workflow

1. Fork the repo
2. Create a feature branch: `git checkout -b feature/my-merciful-addition`
3. Make your changes following the pattern above
4. Run `cargo test` and `cargo clippy`
5. Submit a Pull Request with a clear description of how it increases thriving
6. The lattice will review it through the **DivineLifeBlossomOrchestrator**

All accepted contributions are permanently celebrated in the living lattice.
